Complete Step-by-Step Deployment Guide
Follow these steps in order to build and package your application for AWS:

Step 1: Clean Up Previous Zips (Optional)
If you want to start fresh, delete any old zip files first:

rm -f *.zip
Step 2: Build the Application
This command compiles your frontend and server code into a production-ready dist folder:

npm run build
Step 3: Create a Temporary Workspace
We create a folder to hold only the files that AWS needs. This keeps the final zip file small and clean:

mkdir -p deploy_eb
cp -r dist server shared package.json package-lock.json .ebextensions Procfile deploy_eb/
Step 4: Create the Deployment Zip
AWS requires your configuration files (like Procfile) to be at the "root" level of the zip. To do this correctly, you must enter the folder before zipping:

cd deploy_eb
zip -r ../my_trading_app.zip .
cd ..
Step 5: Final Cleanup
Remove the temporary folder you created in Step 3:

rm -rf deploy_eb
