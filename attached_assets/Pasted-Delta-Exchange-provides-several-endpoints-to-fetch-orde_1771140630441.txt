Delta Exchange provides several endpoints to fetch orders based on different criteria. Here's a comprehensive guide:

Available Endpoints
1. GET /v2/orders - Get Active Orders
Fetches open and pending orders with various filtering options.

2. GET /v2/orders/{order_id} - Get Order by ID
Fetches a specific order using its order ID.

3. GET /v2/orders/client_order_id/{client_oid} - Get Order by Client Order ID
Fetches a specific order using your custom client order ID.

4. GET /v2/orders/history - Get Order History
Fetches cancelled and closed orders.

1. Get Active Orders
Endpoint: GET /v2/orders

Query Parameters:

product_ids (string, optional): Comma-separated product IDs (e.g., "27,5")
states (string, optional): Comma-separated states - "open", "pending", "closed"
contract_types (string, optional): "futures", "perpetual_futures", "call_options", "put_options"
order_types (string, optional): "market", "limit", "stop_market", "stop_limit", "all_stop"
start_time (integer, optional): Start time in microseconds (epoch)
end_time (integer, optional): End time in microseconds (epoch)
after (string, optional): Cursor for pagination (next page)
before (string, optional): Cursor for pagination (previous page)
page_size (integer, optional): Number of records per page (default: 10)
Example Request:


curl -X GET "https://api.india.delta.exchange/v2/orders?product_ids=27&states=open,pending&page_size=50" \
  -H "api-key: your_api_key" \
  -H "timestamp: 1542110948" \
  -H "signature: your_generated_signature" \
  -H "User-Agent: python-rest-client" \
  -H "Content-Type: application/json"

Response Schema:

Properties in Order object:

id (integer): Generated order ID
user_id (integer): Client ID
size (integer): Order size
unfilled_size (integer): Order size not filled yet
side (string): "buy" or "sell"
order_type (string): "limit_order" or "market_order"
limit_price (string): Price level for limit orders
stop_order_type (string): "stop_loss_order" (if applicable)
stop_price (string): Stop price level for stop orders
paid_commission (string): Commission paid for filled order
commission (string): Commission blocked for order
reduce_only (string): "true" or "false" - only closes positions if true
client_order_id (string): Custom user-provided order ID (max 32 length)
state (string): "open", "pending", "closed", or "cancelled"
created_at (string): Unix timestamp in microseconds
product_id (integer): Product ID of the ordered product
product_symbol (string): Product symbol (e.g., "BTCUSD")

4. Get Order History
Endpoint: GET /v2/orders/history

Fetches cancelled and closed orders.

Query Parameters:

product_ids (string, optional): Comma-separated product IDs
contract_types (string, optional): Comma-separated contract types
order_types (string, optional): "market", "limit", "stop_market", "stop_limit", "all_stop"
start_time (integer, optional): Start time in microseconds
end_time (integer, optional): End time in microseconds
after (string, optional): Cursor for pagination
before (string, optional): Cursor for pagination
page_size (integer, optional): Records per page (default: 10)
Example Request:


curl -X GET "https://api.india.delta.exchange/v2/orders/history?product_ids=27&page_size=100" \
  -H "api-key: your_api_key" \
  -H "timestamp: 1542110948" \

Python Implementation Example

import hashlib
import hmac
import requests
import time

base_url = 'https://api.india.delta.exchange'
api_key = 'your_api_key'
api_secret = 'your_api_secret'

def generate_signature(secret, message):
    message = bytes(message, 'utf-8')
    secret = bytes(secret, 'utf-8')
    hash = hmac.new(secret, message, hashlib.sha256)
    return hash.hexdigest()

# Example 1: Get all open orders
method = 'GET'
timestamp = str(int(time.time()))
path = '/v2/orders'
query_string = '?states=open&page_size=50'
payload = ''

signature_data = method + timestamp + path + query_string + payload
signature = generate_signature(api_secret, signature_data)

headers = {
    'api-key': api_key,
    'timestamp': timestamp,
    'signature': signature,
    'User-Agent': 'python-rest-client',
    'Content-Type': 'application/json'
}

url = f'{base_url}{path}'
params = {'states': 'open', 'page_size': 50}

try:
    response = requests.get(url, params=params, headers=headers, timeout=(3, 27))
    response.raise_for_status()
    data = response.json()
    
    if data['success']:
        orders = data['result']
        print(f"Found {len(orders)} open orders")
        for order in orders:
            print(f"Order ID: {order['id']}, Symbol: {order['product_symbol']}, "
                  f"Side: {order['side']}, Size: {order['size']}, Price: {order['limit_price']}")
    else:
        print("Request failed:", data)
        
except requests.exceptions.RequestException as e:
    print(f"Error fetching orders: {e}")

# Example 2: Get specific order by ID
order_id = 123456
method = 'GET'
timestamp = str(int(time.time()))
path = f'/v2/orders/{order_id}'
query_string = ''
payload = ''

headers = {

    'timestamp': timestamp,
    'signature': signature,

    'Content-Type': 'application/json'
}

url = f'{base_url}{path}'

try:

        print(f"Order Details: {order}")
    else:

except requests.exceptions.RequestException as e:
    print(f"Error fetching order: {e}")

Pagination
For endpoints that return multiple orders, use the after and before cursors from the meta object to navigate through pages:

after: Use this cursor to fetch the next page
before: Use this cursor to fetch the previous page
Both become null when no more pages exist in that direction
Common Use Cases
Monitor open orders: Use /v2/orders?states=open
Track specific product: Use /v2/orders?product_ids=27&states=open
Get order history: Use /v2/orders/history
Find order by custom ID: Use /v2/orders/client_order_id/{your_custom_id}
Filter by time range: Use start_time and end_time parameters (in microseconds)
Important Notes
All authenticated requests require proper signature generation (see authentication guide)
Timestamps in responses are in microseconds (not milliseconds)
Use pagination for large result sets to avoid timeouts
The client_order_id is useful for tracking your own orders with custom identifiers
Order states: "open" (active), "pending" (conditional orders), "closed" (filled), "cancelled"
